Hermes is an open-source AI agent from Nous Research that runs in your terminal. With the **xurl** skill loaded, Hermes can post to X, search, pull bookmarks, manage lists, and call any other X API endpoint — through plain English. This guide gets you from zero to a working setup with **Grok** as the model and **xurl** as the bridge to X.

---

## What you need

- A [Grok API key](https://console.x.ai)
- An [X developer account](https://developer.x.com) (you'll create an app and load X API credits in step 4)
- macOS, Linux, WSL2, or Termux (no native Windows)

---

## 1. Install Hermes

```bash
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
source ~/.zshrc    # or ~/.bashrc
```

Verify the install:

```bash
hermes --version
```

<Note>
Full install options (Nix, Docker, Termux) are in the [Hermes installation docs](https://hermes-agent.nousresearch.com/docs/getting-started/installation).
</Note>

---

## 2. Configure Grok as your model provider

Run the wizard and pick **xAI (Grok models — direct API)** when prompted for a provider:

```bash
hermes setup
```

In the wizard:

<Steps>
  <Step title="Pick xAI (Grok models — direct API)">
    Selects Grok as your model backend.
  </Step>
  <Step title="Paste your Grok API key">
    Hermes stores it in `~/.hermes/.env`.
  </Step>
  <Step title="Accept the default base URL">
    `https://api.x.ai/v1` is correct for Grok.
  </Step>
  <Step title="Pick a model">
    `grok-4` is a good default. Change later with `hermes model`.
  </Step>
</Steps>

<Tip>
Copy your key directly from [console.x.ai](https://console.x.ai). PDFs and rich-text editors can substitute lookalike Unicode for ASCII letters and silently break auth.
</Tip>

For everything else (channels, voice, dashboards, tool sandboxing) see the [Hermes quickstart](https://hermes-agent.nousresearch.com/docs/getting-started/quickstart).

## 3. Install xurl

xurl is a standalone CLI for the X API. Hermes shells out to it when you load the xurl skill. Pick whichever install method fits your machine:

<CodeGroup>

```bash Shell script
# No sudo, installs to ~/.local/bin
curl -fsSL https://raw.githubusercontent.com/xdevplatform/xurl/main/install.sh | bash
```

```bash Homebrew
brew install --cask xdevplatform/tap/xurl
```

```bash npm
npm install -g @xdevplatform/xurl
```

```bash Go
go install github.com/xdevplatform/xurl@latest
```

</CodeGroup>

If you used the shell script, make sure `~/.local/bin` is on your PATH:

```bash
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc
```

Confirm:

```bash
xurl --help
```

## 4. Authenticate xurl with the X API

<Warning>
Do this in your terminal directly — never paste secrets into a chat.
</Warning>

### Create the developer app

<Steps>
  <Step title="Open the Developer Console">
    Go to [developer.x.com/en/portal/dashboard](https://developer.x.com/en/portal/dashboard).
  </Step>
  <Step title="Create or reuse an app">
    Create a new app, or reuse an existing one.
  </Step>
  <Step title="Set the redirect URI">
    Under **User authentication settings**, set the redirect URI to `http://localhost:8080/callback`.
  </Step>
  <Step title="Copy the Client ID and Client Secret">
    You'll wire these into xurl in a moment.
  </Step>
</Steps>

### Load API credits

The X API is pay-per-use, so you need to load credits before xurl can make any calls — including `xurl whoami` below.

<Steps>
  <Step title="Open the X API console">
    Go to [console.x.com](https://console.x.com).
  </Step>
  <Step title="Purchase credits">
    Navigate to **Billing → Credits → Purchase Credits** and buy a starting balance.
  </Step>
  <Step title="Set guardrails">
    While you're there, enable **auto-recharge** and set a **max spend cap** — useful insurance when an agent is the one making the calls.
  </Step>
</Steps>

See the [X API pricing docs](/x-api/getting-started/pricing) and the [X API introduction](/x-api/introduction) for current rates and endpoint costs.

### Wire them into xurl and authorize

```bash
xurl auth apps add my-app \
  --client-id YOUR_CLIENT_ID \
  --client-secret YOUR_CLIENT_SECRET

xurl auth oauth2 --app my-app          # opens browser for OAuth
xurl auth default my-app
xurl whoami                            # should print your X handle
```

That's the whole auth dance. xurl now has a token to act on your behalf.

## 5. Use xurl from Hermes

Launch Hermes:

<Tabs>
  <Tab title="Modern TUI (recommended)">
    ```bash
    hermes --tui
    ```
  </Tab>
  <Tab title="Classic CLI">
    ```bash
    hermes
    ```
  </Tab>
</Tabs>

Load the skill with `/xurl`. Hermes will verify xurl is installed and authenticated. Then just describe what you want.

**Post:**

```
post "Hello from Hermes"
```

```
Posted successfully!
- ID: 2047107136023650625
- Link: https://x.com/yourhandle/status/2047107136023650625
```

**Read your bookmarks:**

```
get all of my bookmarks
```

Hermes runs `xurl bookmarks -n 100` and gives you a summarized, readable feed instead of raw JSON.

## 6. Going further: a daily X briefing

Posting and reading is just the surface. The agent gets interesting when it composes xurl with the rest of Hermes' toolbox — Grok for reasoning, built-in TTS, scheduled tasks, and persistent memory. Here's a workflow that turns your morning scroll into a 2-minute audio briefing.

Run it once interactively to see it work:

```
Use xurl to pull my latest 100 home timeline posts, my last 24h of
mentions, and any new replies to my recent posts. Triage what's
actually interesting (skip engagement bait, low-quality replies, repeat
news). Summarize the 5 most important threads in plain English. Then
render the summary as audio to ~/x-briefing.mp3 using TTS.
```

Hermes calls xurl several times in parallel, runs the results through Grok to find the signal, and pipes the summary into Edge TTS. Pop the mp3 into your morning routine and a 20-minute scroll becomes a 2-minute listen.

Once you like the output, schedule it daily with `hermes cron` (see [the cron docs](https://hermes-agent.nousresearch.com/docs)) — same prompt, fully autonomous, every morning. Drop the audio in your morning routine or pipe the transcript to wherever you keep notes.

That's the real story: xurl gives you the X API, but the agent gives you a reason to use it.
