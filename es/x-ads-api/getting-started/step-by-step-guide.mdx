## Cómo obtener acceso a la Ads API

1. Regístrate para obtener una [cuenta de desarrollador](https://console.x.com).
2. Crea una developer App](/fundamentals/developer-apps) y protege tu token.
3. Visita [ads.x.com/help](https://ads.x.com/help) para enviar una solicitud de acceso a la Ads API para cada una de tus developer App(s).

<Note>**Nota:** Si ya estás creando en la X Developer Platform y tienes una cuenta de desarrollador, salta al paso tres.</Note>

### Paso uno: Regístrate para una cuenta de desarrollador

Para realizar una solicitud a cualquiera de los productos de API de X, primero debes registrarte para obtener una cuenta de desarrollador. Dentro de la Developer Console, crea una App y una developer App. Esto te proporcionará un conjunto de credenciales que usarás para autenticar todas las solicitudes a la API.

<Button href="https://console.x.com">
  Regístrate para una cuenta de desarrollador
</Button>

### Paso dos: Guarda las claves y tokens de tu App y mantenlos seguros

Dentro de tu developer App, se te proporcionará un conjunto de API Keys (también conocidas como Consumer Keys). También tendrás la oportunidad de generar un conjunto de Access Tokens que pueden usarse para realizar solicitudes en nombre de tu cuenta personal de X, y un Bearer Token que puede usarse para autenticar endpoints que requieren OAuth 2.0 Bearer Token. Dado que estas claves y tokens no caducan a menos que se regeneren, sugerimos crear variables de entorno o usar un gestor de contraseñas seguro.

También toma nota del App ID, que se puede encontrar en la URL de los detalles de tu App. Esto facilitará la solicitud de acceso a la Ads API en el siguiente paso.

<Note>
**Nota:** Tus claves y tokens solo se mostrarán una vez en la Developer Console, por lo que es importante que almacenes estas credenciales en tu sistema de gestión de contraseñas tan pronto como las generes.

Si pierdes u olvidas las claves y tokens, deberás regenerarlos, lo que crea nuevas claves y tokens, e invalida los anteriores. Esto significa que tendrás que actualizar todas las integraciones que hayas configurado con tus credenciales anteriores.

Aprende más sobre nuestras [mejores prácticas de autenticación](/fundamentals/authentication/guides/authentication-best-practices).
</Note>

### Paso tres: Solicita acceso a la Ads API

En este punto tendrás acceso básico a la X API, pero no a la funcionalidad específica de la X Ads API. A continuación, deberás solicitar y obtener la aprobación de acceso a la Ads API. Visita [ads.x.com/help](https://ads.x.com/help) para enviar una solicitud de acceso a la Ads API para cada una de tus developer App(s).

### Niveles de acceso

Como parte del proceso de solicitud, deberás especificar qué nivel de acceso necesitas. Aprende más sobre [los permisos a nivel de App y a nivel de Ad Account](/x-ads-api/fundamentals/accessing-ads-accounts#levels-of-access).

#### Conversion Only

Acceso a los endpoints de Mobile y Web Conversion con acceso de lectura y escritura.

#### Standard Access

Acceso a los endpoints de Analytics, Campaign Management, Creatives, Custom Audiences y Conversion con acceso de lectura y escritura.

<Note>**Nota:** Después de que tu app sea aprobada para el acceso a la Ads API, deberás [regenerar cualquier user access token](/fundamentals/authentication/oauth-1-0a/obtaining-user-access-tokens) para realizar [solicitudes autenticadas a la Ads API](/x-ads-api/fundamentals/making-authenticated-requests) correctamente.</Note>

<Note>**Nota:** Los desarrolladores de la Ads API que solicitaron acceso antes de julio de 2023 pueden tener diferentes niveles de acceso y permisos, y pueden estar limitados a cinco tokens de OAuth. Consulta nuestra guía sobre [aumentar el acceso](/x-ads-api/getting-started/increasing-access) para acceder a endpoints adicionales o eliminar los límites de tokens para aplicaciones existentes.</Note>

## Realiza tu primera solicitud

Para probar tu acceso a la Ads API, realiza una solicitud al endpoint [GET accounts](/x-ads-api/campaign-management/reference#get-accounts). Este endpoint devolverá las cuentas publicitarias a las que el usuario actualmente autorizado tiene acceso. Usa los IDs de cuenta publicitaria de esta solicitud en solicitudes posteriores a la API para leer y escribir datos de una cuenta publicitaria específica. Usando [Twurl](https://github.com/twitter/twurl) en la línea de comandos, la solicitud se vería de la siguiente manera.

**Solicitud de ejemplo**

```bash
twurl -H ads-api.x.com "/11/accounts"
```

**Respuesta de ejemplo**

