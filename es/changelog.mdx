<Update label="May 4, 2026" description="X API v2">
  ### Migración de los endpoints de búsqueda al nuevo índice de búsqueda

  Hoy migramos los endpoints de búsqueda de X API v2 (`/2/tweets/search/recent`, `/2/tweets/search/all`, `/2/tweets/counts/recent`, `/2/tweets/counts/all`) al nuevo índice de búsqueda de la plataforma principal de X. Esto trae mejoras importantes de rendimiento, una reducción drástica de errores 503 y tres nuevos operadores de precisión: `min\_likes:`, `min\_replies:` y `min\_reposts:`. Nota: los reposts ya no se devuelven en los resultados de búsqueda basados en palabras clave (Filtered Stream no cambia). Más información [aquí](https://devcommunity.x.com/t/important-update-x-api-search-endpoints-migrating-to-new-search-index/264595).
</Update>

<Update label="Apr 16, 2026" description="X API">
  ### Actualización de precios de X API: Owned Reads ahora a $0.001

  A partir del 20 de abril de 2026, introdujimos nuevos precios de "Owned Reads" a $0.001 por recurso (1.000 por $1) para endpoints calificados al acceder a tus propios datos: `GET /2/users/{id}/bookmarks`, `/blocking`, `/muting`, `/pinned_lists`, `/tweets`, `/mentions`, `/liked_tweets`, `/followers`, `/following`, `/owned_lists`, `/followed_lists` y `/list_memberships`. También actualizamos las escrituras — `POST /2/tweets` ahora cuesta $0.015 por publicación, y los Posts que contienen una URL cuestan $0.20 por Post (las respuestas invocadas siguen siendo $0.01). Following, Likes y Quote-Posts a través de la API se han eliminado de todos los niveles de autoservicio. Más información [aquí](https://devcommunity.x.com/t/x-api-pricing-update-owned-reads-now-0-001-other-changes-effective-april-20-2026/263025).
</Update>

<Update label="Mar 20, 2026" description="X API v2">
  ### Obsolescencia del endpoint Account Activity Replay

  Hoy anunciamos la obsolescencia de `POST /2/account_activity/replay/webhooks/{webhook_id}/subscriptions/all`, con efecto a partir del 25 de marzo de 2026 a las 12:00 PM ET. Usa en su lugar el nuevo endpoint consolidado `POST /2/webhooks/replay` — proporciona una funcionalidad idéntica. Más información [aquí](https://devcommunity.x.com/t/deprecation-of-account-activity-replay-endpoint/260354).

<Update label="Mar 18, 2026" description="X API v2">
  ### Filtro de dirección para suscripciones de X Activity API

  Hoy agregamos un filtro opcional `direction` a las suscripciones de X Activity API. Los suscriptores ahora pueden delimitar los eventos de DM y chat a `sent`, `received` o ambos, reduciendo el ruido cuando solo se necesitan eventos salientes o entrantes.
</Update>

<Update label="Mar 13, 2026" description="X API v2">
  ### Actualizaciones del formato de tipos de eventos en X Activity API

  Hoy anunciamos cambios en la nomenclatura de tipos de eventos en X Activity API, con efecto a partir del 18 de marzo de 2026. Los tipos de eventos en PascalCase (por ejemplo, `ProfileBioUpdate`) están siendo descontinuados en favor de un formato con puntos y minúsculas (por ejemplo, `profile.update.bio`). Actualiza cualquier filtrado, declaración switch o lógica de deserialización que dependa de los valores antiguos. Más información [aquí](https://devcommunity.x.com/t/updates-to-event-types-format-in-the-x-activity-api/259765).
</Update>

<Update label="Mar 11, 2026" description="X API v2">
  ### Eventos heredados de Direct Message en X Activity API

  Hoy agregamos eventos heredados de DM (`dm.received`, `dm.sent`, `dm.read`, `dm.indicate_typing`) a la [X Activity API](https://docs.x.com/x-api/activity/introduction). Esto traslada la entrega de eventos de DM desde Account Activity API a la canalización XAA de menor latencia.
</Update>

<Update label="Feb 23, 2026" description="X API v2">
  ### Abordando el spam generado por LLM

  Hoy hicimos cambios para reducir las respuestas automatizadas y de baja calidad en X. Las respuestas programáticas a través de `POST /2/tweets` solo se permiten cuando el autor del Post original ha "invocado" al respondedor (al hacer @mención a esa cuenta o citar uno de sus Posts). Se aplican restricciones adicionales al @mencionar o citar usuarios programáticamente. Estos cambios afectan solo a los niveles de autoservicio — el acceso Enterprise no se ve afectado. Más información [aquí](https://devcommunity.x.com/t/x-api-v2-update-addressing-llm-generated-spam/257909).
</Update>

  Hoy lanzamos la capacidad de editar Posts recientes mediante programación a través de X API. Requisitos: el usuario autenticado debe tener X Premium, el Post debe ser propio y debe haberse creado dentro de la última hora. Consulta los documentos actualizados de [Create or Edit Post](https://docs.x.com/x-api/posts/create-post). <Update label="Sep 19, 2025" description="X API v2">
  ### Connections API para gestionar conexiones de streaming activas

  Hoy lanzamos la [Connections API](https://docs.x.com/x-api/connections/introduction) para gestionar conexiones de streaming activas mediante programación:
  - `GET /2/connections` — lista las conexiones de streaming activas e históricas
  - `DELETE /2/connections/all` — termina todas las conexiones de streaming activas de la aplicación
  - `DELETE /2/connections/{endpoint_id}` — termina conexiones para un endpoint de streaming específico
  - `DELETE /2/connections` — termina conexiones específicas por UUID

  Esto ayuda a los desarrolladores a resolver los errores "TooManyConnections" y limpiar conexiones obsoletas sin esperar tiempos de espera.
</Update>

<Update label="Sep 9, 2025" description="X API v2">
  ### Lanzamiento de Filtered Stream Webhooks API

  Hoy lanzamos la [Filtered Stream Webhooks API](https://docs.x.com/x-api/webhooks/stream/introduction), que entrega Posts públicos coincidentes de forma asíncrona a un webhook registrado en lugar de requerir una conexión persistente. Las reglas se gestionan utilizando los endpoints existentes de `/2/tweets/search/stream/rules`, y los Posts coincidentes (incluyendo el historial de ediciones) se entregan como JSON a tu webhook. Disponible para desarrolladores Enterprise; admite más de 25.000 reglas por proyecto con reglas de hasta 2.048 caracteres. Más información [aquí](https://devcommunity.x.com/t/announcing-the-filtered-stream-webhooks-api/248464).
</Update>

<Update label="Jul 8, 2025" description="X API v2">
  ### Lanzamiento de Community Notes API

  Hoy lanzamos la [Community Notes API](https://docs.x.com/x-api/community-notes/introduction) — un nuevo conjunto de endpoints para que los AI Note Writers participen programáticamente en Community Notes:
  - `GET /2/notes/search/posts_eligible_for_notes` — busca Posts elegibles para recibir una Community Note
  - `GET /2/notes/search/notes_written` — recupera notas escritas previamente por el AI Note Writer autenticado
  - `POST /2/notes` — envía una Community Note

  El acceso requiere inscripción en X Developer AI y aprobación como AI Note Writer de Community Notes.

<Update label="Nov 15, 2021" description="X API v2: Early Access" >
### Actualizaciones importantes de plataforma a X API v2
**Actualizaciones importantes de plataforma: X API v2 es ahora la versión principal, nuevos accesos Essential y Elevated, y cambios de política**

  Hoy anunciamos que X API v2 es ahora la versión principal de X API. Hemos lanzado suficientes endpoints y funcionalidad en X API v2 para satisfacer las necesidades del 90% de todas las Apps existentes construidas sobre X API.

  Estamos lanzando los siguientes cambios hoy para mejorar aún más la experiencia del desarrollador de X API v2:

  * Acceso Essential rápido y gratuito a la API, y acceso Elevated gratuito a los desarrolladores que han tenido sus casos de uso aprobados. Tenemos muchos más detalles sobre estos niveles de acceso disponibles en nuestra página About the X API.

  * Estamos eliminando el lenguaje en nuestra Developer Policy que restringía cómo construyes con las funciones principales de X, y limitaba el número de usuarios que puedes admitir a través de tu app.

    * Estamos abriendo nuestra plataforma para animarte a construir herramientas y productos que hagan X mejor, más saludable y extiendan la conversación pública. [Ver una lista de soluciones que nos encantaría que construyeras](https://developer.x.com/en/docs/twitter-api/what-to-build).

    * Sabemos que construir soluciones que ayuden a las personas en X a menudo significa que un desarrollador tiene que construir (o replicar) algunas de las cosas que están disponibles en X. Estos cambios a nuestra Developer Policy tienen la intención de aportar claridad al ecosistema de desarrolladores y proporcionar una plataforma API abierta que facilite a los desarrolladores construir, innovar y tener un impacto en la conversación pública.

  * [Mapa de endpoints de X API](/x-api/migrate/x-api-endpoint-map)

  * [Acerca de X API](/x-api/getting-started/about-x-api)

<Update label="May 10, 2018" description="X API: Standard v1.1" >
### Cambios en el renderizado de carga útil de Quote
Hoy, estamos agregando un cambio a la forma en que se renderizarán las URLs en la carga útil de quote.

  Estamos agregando una nueva entidad llamada "quoted_status_permalink" para asegurar que el post citado pueda ser referenciado y eliminaremos el enlace t.co del campo "text" del post citado.

  Con el nuevo formato, el enlace t.co ya no se agrega al campo "text" del post citado en el objeto "quoted_status".

  Además, ya no pondremos el url, expanded_url y display_url t.co del post citado en la entidad 'urls'. En cambio, estos se agregarán a un nuevo objeto "quoted_status_permalink" con atributos "url", "expanded" y "display" URL. Esto significa que el objeto "quoted_status_permalink" se reservará para el enlace t.co al post citado, mientras que el array "entities.urls" se puede usar para identificar cualquier enlace compartido dentro del post original o quote.

  Lee nuestra [publicación del foro](https://devcommunity.x.com/t/updating-how-urls-are-rendered-in-the-quote-tweet-payload/105473) y la [documentación actualizada](https://devcommunity.x.com/t/upcoming-changes-to-the-developer-platform/104603) para obtener más detalles.
</Update>
