The article body is described as a `content_state` object — a block-based document model that can feel unfamiliar at first. This guide gives you a copy-paste builder utility (Python and JavaScript) that handles all the offset arithmetic, plus common recipes for every formatting feature.

---

## How the content model works

Think of `content_state` like a document made of **stacked rows**:

- **Each block** = one row (a paragraph, heading, list item, blockquote, or embedded element)
- **`text`** = the plain text of that row, exactly as it would be read aloud
- **Inline styles** (bold, italic) = character ranges applied on top of the plain text
- **Entities** (links, embedded posts) = named objects that blocks reference by key

```
┌─────────────────────────────────────────────────────────────┐
│ Block 1: type=header-one  │ text="Why X Articles Matter"   │
├─────────────────────────────────────────────────────────────┤
│ Block 2: type=unstyled    │ text="X Articles let you…"     │
│                           │ inline_style_ranges: bold[0,1] │
├─────────────────────────────────────────────────────────────┤
│ Block 3: type=atomic      │ (embedded post — no text)      │
│                           │ entity_ranges → entity key 0   │
└─────────────────────────────────────────────────────────────┘

entities: [{ key:"0", value: { type:"post", data:{post_id:"…"} } }]
```

The trickiest part is **offset arithmetic** for inline styles: every bold/italic/link range is described as `{ offset, length }` — the character position within the block's `text` string where the style starts, and how many characters it covers. The builder below calculates all of that for you.

## The ArticleBuilder utility

Copy either version into your project and use it to compose your `content_state`. Both support full method chaining.

<CodeGroup>

```python Python
import re

class ArticleBuilder:
    """Builds a content_state payload for the X Articles API."""

    def __init__(self):
        self._blocks = []
        self._entities = []
        self._next_key = 0

    # ── private helpers ──────────────────────────────────────────────

    def _add_block(self, block):
        self._blocks.append(block)
        return self  # allows chaining

    def _register_entity(self, value):
        key = self._next_key
        self._next_key += 1
        self._entities.append({"key": str(key), "value": value})
        return key

    # ── structural blocks ────────────────────────────────────────────

    def paragraph(self, text):
        """Plain paragraph."""
        return self._add_block({"text": text, "type": "unstyled"})

    def heading(self, text, level=1):
        """Heading — level 1 (H1), 2 (H2), or 3 (H3)."""
        types = {1: "header-one", 2: "header-two", 3: "header-three"}
        return self._add_block({"text": text, "type": types.get(level, "header-one")})

    def blockquote(self, text):
        """Blockquote."""
        return self._add_block({"text": text, "type": "blockquote"})

    def bullet(self, text):
        """Unordered list item."""
        return self._add_block({"text": text, "type": "unordered-list-item"})

    def ordered_item(self, text):
        """Ordered list item."""
        return self._add_block({"text": text, "type": "ordered-list-item"})

    # ── inline formatting ────────────────────────────────────────────

    def styled(self, segments):
        """
        Paragraph with inline bold / italic / strikethrough styling.

        segments: list of (text, style) tuples where style is one of:
            None                        — no style (plain text)
            "bold"
            "italic"
            "strikethrough"
            ["bold", "italic"]          — apply multiple styles at once

        Example:
            builder.styled([
                ("This article covers ", None),
                ("three key points", "bold"),
                (" you should know.", None),
            ])
        """
        full_text = ""
        ranges = []

        for (text, style) in segments:
            offset = len(full_text)
            full_text += text
            if style:
                for s in ([style] if isinstance(style, str) else style):
                    ranges.append({"offset": offset, "length": len(text), "style": s})

        block = {"text": full_text, "type": "unstyled"}
        if ranges:
            block["inline_style_ranges"] = ranges
        return self._add_block(block)

    # ── mentions & hashtags ──────────────────────────────────────────

    def mentions(self, text):
        """
        Paragraph that auto-detects @mentions and #hashtags.

        Example:
            builder.mentions("Thanks @xdevelopers for shipping #XAPI!")
        """
        detected_mentions = [
            {"from_index": m.start(), "to_index": m.end(), "text": m.group(1)}
            for m in re.finditer(r"@(\w+)", text)
        ]
        detected_hashtags = [
            {"from_index": m.start(), "to_index": m.end(), "text": m.group(1)}
            for m in re.finditer(r"#(\w+)", text)
        ]

        block = {"text": text, "type": "unstyled"}
        if detected_mentions or detected_hashtags:
            block["data"] = {}
            if detected_mentions:
                block["data"]["mentions"] = detected_mentions
            if detected_hashtags:
                block["data"]["hashtags"] = detected_hashtags
        return self._add_block(block)

    # ── hyperlinks ───────────────────────────────────────────────────

    def link(self, link_text, url, *, before="", after=""):
        """
        Paragraph containing a clickable hyperlink.

        before / after: optional plain text surrounding the linked text.

        Example:
            builder.link(
                "X Developer Platform",
                url="https://developer.x.com",
                before="Check out the ",
                after=" for the full API reference.",
            )
        """
        full_text = before + link_text + after
        offset = len(before)
        key = self._register_entity({
            "type": "link",
            "mutability": "mutable",
            "data": {"url": url},
        })
        return self._add_block({
            "text": full_text,
            "type": "unstyled",
            "entity_ranges": [{"offset": offset, "length": len(link_text), "key": key}],
            "data": {
                "urls": [
                    {"from_index": offset, "to_index": offset + len(link_text), "text": url}
                ]
            },
        })

    # ── embedded posts ───────────────────────────────────────────────

    def embed_post(self, post_id):
        """
        Embed another X Post inline.

        The post is rendered as an atomic block (a full post card).
        Wrap it with paragraph() calls for intro/outro text.

        Example:
            builder.paragraph("Here's a post worth reading:")
            builder.embed_post("1895586542058820000")
            builder.paragraph("What do you think?")
        """
        key = self._register_entity({
            "type": "post",
            "mutability": "immutable",
            "data": {"post_id": post_id},
        })
        return self._add_block({
            "text": " ",
            "type": "atomic",
            "entity_ranges": [{"offset": 0, "length": 1, "key": key}],
        })

    # ── output ───────────────────────────────────────────────────────

    def build(self):
        """Return the content_state dict, ready to pass to the API."""
        return {"blocks": self._blocks, "entities": self._entities}
```

```javascript JavaScript
class ArticleBuilder {
  constructor() {
    this._blocks = [];
    this._entities = [];
    this._nextKey = 0;
  }

  // ── private helpers ──────────────────────────────────────────────

  _addBlock(block) {
    this._blocks.push(block);
    return this; // allows chaining
  }

  _registerEntity(value) {
    const key = this._nextKey++;
    this._entities.push({ key: String(key), value });
    return key;
  }

  // ── structural blocks ────────────────────────────────────────────

  paragraph(text) {
    return this._addBlock({ text, type: "unstyled" });
  }

  heading(text, level = 1) {
    const types = { 1: "header-one", 2: "header-two", 3: "header-three" };
    return this._addBlock({ text, type: types[level] ?? "header-one" });
  }

  blockquote(text) {
    return this._addBlock({ text, type: "blockquote" });
  }

  bullet(text) {
    return this._addBlock({ text, type: "unordered-list-item" });
  }

  orderedItem(text) {
    return this._addBlock({ text, type: "ordered-list-item" });
  }

  // ── inline formatting ────────────────────────────────────────────

  styled(segments) {
    /**
     * Paragraph with inline bold / italic / strikethrough styling.
     *
     * segments: Array of [text, style] pairs. style can be:
     *   null                     — plain text
     *   "bold" | "italic" | "strikethrough"
     *   ["bold", "italic"]       — multiple styles at once
     *
     * Example:
     *   builder.styled([
     *     ["This article covers ", null],
     *     ["three key points", "bold"],
     *     [" you should know.", null],
     *   ]);
     */
    let fullText = "";
    const ranges = [];

    for (const [text, style] of segments) {
      const offset = fullText.length;
      fullText += text;
      if (style) {
        const styles = Array.isArray(style) ? style : [style];
        for (const s of styles) {
          ranges.push({ offset, length: text.length, style: s });
        }
      }
    }

    const block = { text: fullText, type: "unstyled" };
    if (ranges.length) block.inline_style_ranges = ranges;
    return this._addBlock(block);
  }

  // ── mentions & hashtags ──────────────────────────────────────────

  mentions(text) {
    /**
     * Paragraph that auto-detects @mentions and #hashtags.
     *
     * Example:
     *   builder.mentions("Thanks @xdevelopers for shipping #XAPI!");
     */
    const detectedMentions = [...text.matchAll(/@(\w+)/g)].map((m) => ({
      from_index: m.index,
      to_index: m.index + m[0].length,
      text: m[1],
    }));
    const detectedHashtags = [...text.matchAll(/#(\w+)/g)].map((m) => ({
      from_index: m.index,
      to_index: m.index + m[0].length,
      text: m[1],
    }));

    const block = { text, type: "unstyled" };
    if (detectedMentions.length || detectedHashtags.length) {
      block.data = {};
      if (detectedMentions.length) block.data.mentions = detectedMentions;
      if (detectedHashtags.length) block.data.hashtags = detectedHashtags;
    }
    return this._addBlock(block);
  }

  // ── hyperlinks ───────────────────────────────────────────────────

  link(linkText, url, { before = "", after = "" } = {}) {
    /**
     * Paragraph containing a clickable hyperlink.
     *
     * before / after: optional plain text surrounding the linked text.
     *
     * Example:
     *   builder.link(
     *     "X Developer Platform",
     *     "https://developer.x.com",
     *     { before: "Check out the ", after: " for the full API reference." }
     *   );
     */
    const fullText = before + linkText + after;
    const offset = before.length;
    const key = this._registerEntity({
      type: "link",
      mutability: "mutable",
      data: { url },
    });

    return this._addBlock({
      text: fullText,
      type: "unstyled",
      entity_ranges: [{ offset, length: linkText.length, key }],
      data: {
        urls: [
          { from_index: offset, to_index: offset + linkText.length, text: url },
        ],
      },
    });
  }

  // ── embedded posts ───────────────────────────────────────────────

  embedPost(postId) {
    /**
     * Embed another X Post inline as a full post card.
     *
     * Example:
     *   builder.paragraph("Here's a post worth reading:");
     *   builder.embedPost("1895586542058820000");
     *   builder.paragraph("What do you think?");
     */
    const key = this._registerEntity({
      type: "post",
      mutability: "immutable",
      data: { post_id: postId },
    });
    return this._addBlock({
      text: " ",
      type: "atomic",
      entity_ranges: [{ offset: 0, length: 1, key }],
    });
  }

  // ── output ───────────────────────────────────────────────────────

  build() {
    /** Return the content_state object, ready to pass to the API. */
    return { blocks: this._blocks, entities: this._entities };
  }
}
```

### Plain text article

The simplest possible article — paragraphs only.

<CodeGroup>

```python Python
builder = ArticleBuilder()
builder.paragraph("This is my first paragraph.")
builder.paragraph("And this is my second paragraph. Keep going as long as you like.")

payload = {
    "title": "My Plain Article",
    "content_state": builder.build(),
}
```

```javascript JavaScript
const builder = new ArticleBuilder();
builder
  .paragraph("This is my first paragraph.")
  .paragraph("And this is my second paragraph. Keep going as long as you like.");

const payload = {
  title: "My Plain Article",
  content_state: builder.build(),
};
```

### Cover media

`cover_media` is a top-level field on the draft payload, not part of `content_state`. Upload your image first using the [Media Upload endpoint](/x-api/media/quickstart/media-upload-chunked), then reference the `media_id`.

<CodeGroup>

```python Python
import requests

USER_ACCESS_TOKEN = "YOUR_USER_ACCESS_TOKEN"

builder = ArticleBuilder()
builder.heading("A Visual Article", level=1)
builder.paragraph("This article has a cover image.")

payload = {
    "title": "A Visual Article",
    "content_state": builder.build(),
    "cover_media": {
        "media_category": "TWEET_IMAGE",
        "media_id": "1234567890123456789",   # from the Media Upload API
    },
}

response = requests.post(
    "https://api.x.com/2/article/draft",
    headers={
        "Authorization": f"Bearer {USER_ACCESS_TOKEN}",
        "Content-Type": "application/json",
    },
    json=payload,
)
print(response.json())
```

```javascript JavaScript
const USER_ACCESS_TOKEN = "YOUR_USER_ACCESS_TOKEN";

const builder = new ArticleBuilder();
builder.heading("A Visual Article", 1).paragraph("This article has a cover image.");

  method: "POST",

    Authorization: `Bearer ${USER_ACCESS_TOKEN}`,

  },
  body: JSON.stringify(payload),
});
console.log(await response.json());
```

## Content model reference

### Block types

| `type` value | Description |
|:-------------|:------------|
| `unstyled` | Plain paragraph (default block type) |
| `header-one` | H1 heading |
| `header-two` | H2 heading |
| `header-three` | H3 heading |
| `unordered-list-item` | Bullet list item |
| `ordered-list-item` | Numbered list item |
| `blockquote` | Blockquote |
| `atomic` | Embedded element (post, image) — `text` must be a single space `" "` |

### Inline styles

Applied via `inline_style_ranges: [{ offset, length, style }]` on a block.

| `style` value | Description |
|:--------------|:------------|
| `bold` | **Bold** text |
| `italic` | *Italic* text |
| `strikethrough` | ~~Struck-through~~ text |

### Entity types

Entities are registered in the top-level `entities` array and referenced from blocks via `entity_ranges`.

| `type` value | `mutability` | Description |
|:-------------|:-------------|:------------|
| `link` | `mutable` | Hyperlink — `data.url` holds the destination URL |
| `post` | `immutable` | Embedded X Post — `data.post_id` holds the Post ID |
| `image` | `immutable` | Embedded image — see media upload docs for `data` shape |

### Understanding offsets

Offset values are **zero-based character positions** within a block's `text` string:

```
text:   "Hello, bold world!"
index:   0123456789...

{ "offset": 7, "length": 4, "style": "bold" }
  → characters 7–10 = "bold"
```

For `data.mentions`, `data.hashtags`, and `data.urls`, `from_index` is inclusive and `to_index` is exclusive (Python slice convention). The `ArticleBuilder.styled()`, `link()`, and `mentions()` methods calculate all of these for you automatically.
