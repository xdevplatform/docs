## Endpoint

```
POST https://api.x.com/2/articles/{article_id}/publish
```

---

## Authentication

| Type | Detail |
|:-----|:-------|
| OAuth 2.0 User Context | Required — must be the same user who created the draft |
| Scopes | `tweet.read`, `tweet.write`, `users.read` |
| Subscription | The authenticated user must have an active **X Premium (Blue Verified)** subscription |

---

## Request

### Path parameters

| Parameter | Type | Required | Description |
|:----------|:-----|:---------|:------------|
| `article_id` | string | Yes | The `id` returned from [`POST /2/article/draft`](/x-api/articles/create-draft-article) |

### Headers

| Header | Value | Required |
|:-------|:------|:---------|
| `Authorization` | `Bearer <user_access_token>` | Yes |

### Body

No request body — the endpoint takes no payload, only the path parameter.

## Example request

<CodeGroup dropdown>

```bash cURL
ARTICLE_ID="2030915708830224384"

curl -X POST "https://api.x.com/2/articles/${ARTICLE_ID}/publish" \
  -H "Authorization: Bearer $USER_ACCESS_TOKEN"
```

```python Python
import requests

USER_ACCESS_TOKEN = "YOUR_USER_ACCESS_TOKEN"
ARTICLE_ID = "2030915708830224384"

response = requests.post(
    f"https://api.x.com/2/articles/{ARTICLE_ID}/publish",
    headers={"Authorization": f"Bearer {USER_ACCESS_TOKEN}"},
)
print(response.json())
```


## Response

### 200 OK

| Field | Type | Description |
|:------|:-----|:------------|
| `data.post_id` | string | The ID of the X Post that was automatically created when the article was published. Use this with the [Posts API](/x-api/posts/get-post-by-id) to look up or engage with the article's Post. |

```json
{
  "data": {
    "post_id": "2030916179745800224"
  }
}
```

## Error responses

<AccordionGroup>
  <Accordion title="403 Forbidden — Premium subscription required">
    ```json
    {
      "errors": [{
        "message": "You are not authorized to create or publish articles. A Premium subscription is required.",
        "code": 403
      }]
    }
    ```
    The authenticated user does not have an active X Premium (Blue Verified) subscription.
  </Accordion>

  <Accordion title="400 Bad Request — article not found or not owned">
    ```json
    {
      "errors": [{
        "message": "The specified article could not be found or is not owned by the authenticated user.",
        "code": 400
      }]
    }
    ```
    The `article_id` does not correspond to a draft owned by the authenticated user. Verify that you are using the `id` returned from [`POST /2/article/draft`](/x-api/articles/create-draft-article) with the same user token.
  </Accordion>

  <Accordion title="401 Unauthorized">
    ```json
    { "title": "Unauthorized", "status": 401, "detail": "Unauthorized" }
    ```
    The access token is missing, expired, or was not issued with the required scopes. Ensure you are using a **User Access Token** — App-only Bearer Tokens are not accepted.
  </Accordion>
</AccordionGroup>
