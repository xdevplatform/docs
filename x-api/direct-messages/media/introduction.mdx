The DM Media endpoint lets you download media attachments (images, videos, GIFs) from Direct Message events using **OAuth 2.0**.

## Overview

When a DM event contains a media attachment, the API returns a `media_url` field that looks like a TON (Twitter Object Nest) URL:

```
https://ton.twitter.com/1.1/ton/data/dm/1034828552951160836/1034828533812486145/oP5p359h.jpg
```

Previously, fetching media from this URL required constructing a signed **OAuth 1.0a** request — a complex, token-dependent operation. The new `/2/dm_conversations/media` endpoint exposes a simpler, modern path to download the same media using an **OAuth 2.0** user access token.

---

## How it works

The three path segments at the end of any TON URL map directly to parameters in the new endpoint:

| TON URL segment | New endpoint parameter |
|:----------------|:----------------------|
| `1034828552951160836` | `{dm_id}` |
| `1034828533812486145` | `{media_id}` |
| `oP5p359h.jpg` | `{resource_id}` |

**TON URL:**
```
https://ton.twitter.com/1.1/ton/data/dm/{dm_id}/{media_id}/{resource_id}
```

**New endpoint:**
```
https://api.x.com/2/dm_conversations/media/{dm_id}/{media_id}/{resource_id}
```

---

## Endpoint

| Method | Endpoint |
|:-------|:---------|
| GET | `/2/dm_conversations/media/{dm_id}/{media_id}/{resource_id}` |

---

## Authentication

<Warning>
This endpoint requires **OAuth 2.0 User Context** authentication. The authenticated user must be a participant in the DM conversation that contains the media.
</Warning>

| Method | Supported |
|:-------|:----------|
| OAuth 2.0 Authorization Code with PKCE | ✔ Recommended |
| OAuth 1.0a User Context | ✗ Not supported — use the legacy TON URL method instead |
| App-Only (Bearer Token) | ✗ Not supported |

### Required OAuth 2.0 scopes

| Scope | Purpose |
|:------|:--------|
| `dm.read` | Access to the user's DM conversations |
| `tweet.read` | Required alongside `dm.read` |
| `users.read` | Required alongside `dm.read` |

## Getting started

<Note>
**Prerequisites**

- An approved [developer account](https://developer.x.com/en/portal/petition/essential/basic-info)
- A [Project and App](/resources/fundamentals/developer-apps) in the Developer Console
- A user OAuth 2.0 access token obtained via [Authorization Code with PKCE](/resources/fundamentals/authentication/oauth-2-0/authorization-code) with `dm.read`, `tweet.read`, and `users.read` scopes
</Note>
