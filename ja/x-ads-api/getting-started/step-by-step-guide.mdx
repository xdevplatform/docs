## Ads API へのアクセス方法

1. [開発者アカウント](https://console.x.com)に登録します。
2. developer App](/fundamentals/developer-apps) を作成し、トークンを安全に保管します。
3. [ads.x.com/help](https://ads.x.com/help) にアクセスして、各 developer App に対する Ads API アクセスのリクエストを送信します。

<Note>**注:** すでに X Developer Platform 上で開発を行っており、開発者アカウントをお持ちの場合は、ステップ 3 にスキップしてください。</Note>

### ステップ 1: 開発者アカウントへの登録

X の API プロダクトにリクエストを送信するには、まず開発者アカウントに登録する必要があります。Developer Console 内で App および developer App を作成します。これにより、API へのすべてのリクエストの認証に使用する一連のクレデンシャルが提供されます。

<Button href="https://console.x.com">
  開発者アカウントに登録する
</Button>

### ステップ 2: App のキーとトークンを安全に保管する

developer App 内では、API キー(Consumer Keys とも呼ばれます)のセットが提供されます。また、ご自身の X アカウントの代理としてリクエストを行うために使用できる Access Tokens のセットや、OAuth 2.0 Bearer Token を必要とするエンドポイントの認証に使用できる Bearer Token を生成することもできます。これらのキーとトークンは再生成しない限り有効期限が切れないため、環境変数を作成するか、安全なパスワードマネージャーを使用することを推奨します。

また、App ID は App 詳細の URL に表示されるので、メモしておいてください。次のステップで Ads API へのアクセスをリクエストする際に役立ちます。

<Note>
**注:** キーとトークンは Developer Console に一度しか表示されないため、生成後はすぐにパスワード管理システムに保存することが重要です。

キーとトークンを紛失または忘れた場合は、再生成が必要となり、新しいキーとトークンが作成されて古いものは無効になります。つまり、以前のクレデンシャルで設定したすべての連携を更新する必要があります。

[認証のベストプラクティス](/fundamentals/authentication/guides/authentication-best-practices)について詳しく学べます。
</Note>

### ステップ 3: Ads API へのアクセスを申請する

この時点で X API への基本的なアクセスは可能になりますが、X Ads API 固有の機能にはアクセスできません。次に、Ads API アクセスをリクエストし、承認される必要があります。[ads.x.com/help](https://ads.x.com/help) にアクセスして、各 developer App に対する Ads API アクセスのリクエストを送信してください。

### アクセスティア

申請プロセスの一環として、必要なアクセスレベルを指定する必要があります。[App レベルおよび Ad Account レベルの権限](/x-ads-api/fundamentals/accessing-ads-accounts#levels-of-access)について詳しく学べます。

#### Conversion Only

Mobile および Web Conversion エンドポイントへの読み取り・書き込みアクセス。

#### Standard Access

Analytics、Campaign Management、Creatives、Custom Audiences、および Conversion エンドポイントへの読み取り・書き込みアクセス。

<Note>**注:** アプリが Ads API アクセスの承認を受けた後、適切に[認証された Ads API リクエスト](/x-ads-api/fundamentals/making-authenticated-requests)を行うために、[ユーザーアクセストークンを再生成](/fundamentals/authentication/oauth-1-0a/obtaining-user-access-tokens)する必要があります。</Note>

<Note>**注:** 2023 年 7 月以前にアクセスをリクエストした Ads API 開発者は、異なるレベルのアクセスや権限を持っている場合があり、OAuth トークンが 5 つまでに制限されている場合があります。既存のアプリケーションで追加のエンドポイントへのアクセスやトークン制限の引き上げが必要な場合は、[アクセスの拡張](/x-ads-api/getting-started/increasing-access)に関するガイドをご覧ください。</Note>

## 最初のリクエストを送る

Ads API へのアクセスをテストするには、[GET accounts](/x-ads-api/campaign-management/reference#get-accounts) エンドポイントにリクエストを送ります。このエンドポイントは、現在認証されたユーザーがアクセス可能な広告アカウントを返します。このリクエストで得られた広告アカウント ID を、後続の API リクエストで使用して、特定の広告アカウントのデータを読み書きします。コマンドラインで [Twurl](https://github.com/twitter/twurl) を使用する場合、リクエストは次のようになります。

**リクエスト例**

```bash
twurl -H ads-api.x.com "/11/accounts"
```

**レスポンス例**

