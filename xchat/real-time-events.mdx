The X Activity API delivers encrypted chat events (`chat.received` and `chat.sent`) to your application in real-time. Combined with webhooks, you can build responsive chat applications that react instantly to new messages.

## Overview

When a user sends or receives an encrypted DM, X can deliver the event to your webhook endpoint within milliseconds. The event payload contains the encrypted message, which you decrypt using the [Chat XDK](/xchat/xchat-xdk).

```mermaid actions={false}
flowchart LR
    A["User sends<br/>encrypted DM"] --> B["X Activity API"] --> C["Your webhook"] --> D["Chat XDK<br/>decrypts message"]
```

## Chat Event Types

| Event Name | Description | When it fires |
|:-----------|:------------|:--------------|
| `chat.received` | User receives an encrypted DM | When someone sends your subscribed user a message |
| `chat.sent` | User sends an encrypted DM | When your subscribed user sends a message |
| `chat.conversation_join` | User joins a group conversation | When your subscribed user is added to a group chat |

<Note>
Chat events are **private events** — they require OAuth 2.0 authentication from the user before you can subscribe to their events.
</Note>

## Quick Setup

### 1. Register a Webhook

First, register a webhook endpoint to receive events:

<CodeGroup>
```python Python
from xdk import Client

client = Client(bearer_token="YOUR_BEARER_TOKEN")

# Register your webhook URL
webhook = client.webhooks.create({"url": "https://your-app.com/webhook"})
print(f"Webhook ID: {webhook.id}")
```

```typescript TypeScript
import { Client } from '@xdevplatform/xdk';

const client = new Client({ bearerToken: 'YOUR_BEARER_TOKEN' });

// Register your webhook URL
const webhook = await client.webhooks.create({
  url: 'https://your-app.com/webhook'
});
console.log(`Webhook ID: ${webhook.id}`);
```
</CodeGroup>

### 2. Implement CRC Validation

X sends periodic Challenge-Response Check (CRC) requests to verify you control the webhook endpoint. Your server must respond with an HMAC-SHA256 hash.

<CodeGroup>
```python Python (Flask)
import hmac
import hashlib
import base64
from flask import Flask, request, jsonify

app = Flask(__name__)
CONSUMER_SECRET = "your_consumer_secret"

@app.route('/webhook', methods=['GET'])
def crc_check():
    crc_token = request.args.get('crc_token')
    
    # Create HMAC SHA-256 hash
    sha256_hash = hmac.new(
        CONSUMER_SECRET.encode('utf-8'),
        msg=crc_token.encode('utf-8'),
        digestmod=hashlib.sha256
    ).digest()
    
    response_token = base64.b64encode(sha256_hash).decode('utf-8')
    
    return jsonify({
        'response_token': f'sha256={response_token}'
    })
```

```typescript TypeScript (Express)
import express from 'express';
import crypto from 'crypto';

const app = express();
const CONSUMER_SECRET = 'your_consumer_secret';

app.get('/webhook', (req, res) => {
  const crcToken = req.query.crc_token as string;
  
  // Create HMAC SHA-256 hash
  const hash = crypto
    .createHmac('sha256', CONSUMER_SECRET)
    .update(crcToken)
    .digest('base64');
  
  res.json({
    response_token: `sha256=${hash}`
  });
});
```
</CodeGroup>

### 3. Subscribe to Chat Events

Create a subscription for the user's chat events:

<CodeGroup>
```python Python
# Subscribe to chat events for a user (one subscription per event type)
received_sub = client.activity.create_subscription(
    {"event_type": "chat.received", "filter": {"user_id": "USER_ID_TO_MONITOR"}}
)
print(f"Subscription: {received_sub.data}")

sent_sub = client.activity.create_subscription(
    {"event_type": "chat.sent", "filter": {"user_id": "USER_ID_TO_MONITOR"}}
)
print(f"Subscription: {sent_sub.data}")
```

```typescript TypeScript
// Subscribe to chat events for a user (one subscription per event type)
const receivedSub = await client.activity.createSubscription({
  event_type: 'chat.received',
  filter: { user_id: 'USER_ID_TO_MONITOR' }
```

### 4. Handle Incoming Events

When a chat event arrives, decrypt it using the Chat XDK. The encrypted message is in `payload.encoded_event`; the conversation key (when attached) is in `payload.conversation_key_change_event`.

<CodeGroup>
```python Python (Flask)
from chat_xdk import Chat

# Initialize Chat XDK (unlocked at startup — see Getting Started)
chat = Chat(JUICEBOX_CONFIG)
chat.unlock("YOUR_PIN")

# Cache raw conversation keys in memory (use a database in production)
# Shape: { conversation_id: { version: key_bytes } }
conversation_keys = {}

@app.route('/webhook', methods=['POST'])
def handle_event():
    event = request.json
    data = event.get('data', {})

    if data.get('event_type') in ['chat.received', 'chat.sent']:
        payload = data.get('payload', {})
        encoded_event = payload.get('encoded_event')
        conversation_id = payload.get('conversation_id')
        sender_id = payload.get('sender_id')

        # Refresh keys from the attached key change event (new conversation or rotation)
        key_change = payload.get('conversation_key_change_event')
        if key_change:
            extracted = chat.extract_conversation_keys([key_change])
            conversation_keys[conversation_id] = extracted['keys']

        # Get stored conversation keys, then decrypt (pass sender signing keys to verify)
        conv_keys = conversation_keys.get(conversation_id, {})
        event_obj = chat.decrypt_event(encoded_event, conv_keys, get_signing_keys(sender_id))

        if event_obj.get('type') == 'Message':
            content = event_obj.get('content', {})
            if content.get('content_type') == 'Text':
                print(f"New message: {content.get('text')}")
                print(f"From: {event_obj.get('sender_id')}")
                print(f"Verified: {event_obj.get('verified')}")

                # Process the message (e.g., respond, store, analyze)
                handle_incoming_message(event_obj, conversation_id)

    return '', 200
```

```typescript TypeScript (Express)
import { createChat } from 'chat-xdk';

// Initialize Chat XDK (unlocked at startup — see Getting Started)
const chat = await createChat({ juiceboxConfig, getAuthToken });
await chat.unlock('YOUR_PIN');

// Cache raw conversation keys in memory (use a database in production)
const conversationKeys = new Map<string, Record<string, Uint8Array>>();

app.post('/webhook', express.json(), async (req, res) => {
  const data = req.body?.data ?? {};

  if (data.event_type === 'chat.received' || data.event_type === 'chat.sent') {
    const payload = data.payload || {};
    const encodedEvent = payload.encoded_event;
    const conversationId = payload.conversation_id;
    const senderId = payload.sender_id;

    // Refresh keys from the attached key change event (new conversation or rotation)
    if (payload.conversation_key_change_event) {
      const extracted = chat.extractConversationKeys([payload.conversation_key_change_event]);
      conversationKeys.set(conversationId, extracted.keys);
    }

    // Get stored conversation keys, then decrypt (pass sender signing keys to verify)
    const convKeys = conversationKeys.get(conversationId) || {};
    const eventObj = chat.decryptEvent(encodedEvent, convKeys, await getSigningKeys(senderId));

    if (eventObj.type === 'message') {
      const content = eventObj.content;
      if (content?.contentType === 'text') {
        console.log('New message:', content.text);
        console.log('From:', eventObj.senderId);
        console.log('Verified:', eventObj.verified);

        // Process the message
        await handleIncomingMessage(eventObj, conversationId);
      }
    }
  }

  res.sendStatus(200);
});
```
</CodeGroup>

<Note>
**Decrypted event casing differs by SDK.** The Python and Rust bindings use PascalCase event types and snake_case fields (`"Message"`, `content_type`, `sender_id`); the JavaScript/WASM binding uses camelCase throughout (`"message"`, `contentType`, `senderId`). The examples above reflect each language's convention.
</Note>

## Event Payload Structure

Chat events are delivered as JSON wrapped in a `data` envelope. The structure is the same for **webhooks** and the **activity stream** (`GET /2/activity/stream`):

```json
{
  "data": {
    "event_uuid": "abc123def456",
    "event_type": "chat.received",
    "filter": { "user_id": "1234567890" },
    "tag": "optional subscription tag",
    "payload": {
      "conversation_id": "1234567890-0987654321",
      "conversation_token": "#EN4d6KF...",
      "sender_id": "0987654321",
      "id": "2046813948905676800",
      "created_at_msec": "1776833448040",
      "encoded_event": "base64_encoded_message_event...",
      "conversation_key_version": "1765441245499",
      "conversation_key_change_event": "base64_encoded_key_change_event...",
      "message_event_signature": {
        "public_key_version": "1765441245499",
        "signature": "base64...",
        "signature_version": "4",
        "signing_public_key": "base64..."
      }
    }
  }
}
```

| Field | Description |
|:------|:------------|
| `data.event_type` | `chat.received`, `chat.sent`, or `chat.conversation_join` |
| `data.event_uuid` | Unique event ID — use it to deduplicate retried deliveries |
| `data.filter` | The subscription filter that matched (e.g. `user_id`) |
| `data.payload.conversation_id` | The conversation identifier (group IDs are prefixed with `g`) |
| `data.payload.sender_id` | User ID of the message sender |
| `data.payload.id` | The message ID |
| `data.payload.encoded_event` | Base64-encoded message event — **decrypt this** with the Chat XDK |
| `data.payload.conversation_key_version` | Version of the conversation key needed to decrypt `encoded_event` |
| `data.payload.conversation_key_change_event` | Present when a new key is needed; pass it to `extract_conversation_keys` |
| `data.payload.encrypted_conversation_key` | Optional — the conversation key encrypted directly for you (decrypt with `decrypt_conversation_key`) |
| `data.payload.conversation_token` | Optional conversation token |
| `data.payload.message_event_signature` | Optional signature metadata for the event |

<Warning>
Earlier drafts of these docs referred to `payload.encrypted_event` and `payload.key_change`. The current field names are **`encoded_event`** (the message to decrypt) and **`conversation_key_change_event`** (the key event to feed into `extract_conversation_keys`).
</Warning>

## Webhook Requirements

| Requirement | Description |
|:------------|:------------|
| **HTTPS** | Webhook URL must use HTTPS |
| **Publicly accessible** | URL must be reachable from the internet |
| **No port in URL** | URLs like `https://example.com:5000/webhook` won't work |
| **Fast response** | Respond within 10 seconds |
| **200 OK** | Return 200 status to acknowledge receipt |
| **CRC support** | Respond to Challenge-Response Check requests |

## Signature Verification

Each webhook POST includes an `x-twitter-webhooks-signature` header. Verify this signature to confirm X is the source:

<CodeGroup>
```python Python
import hmac
import hashlib
import base64

def verify_signature(payload: bytes, signature: str, consumer_secret: str) -> bool:
    expected = 'sha256=' + base64.b64encode(
        hmac.new(
            consumer_secret.encode('utf-8'),
            msg=payload,
            digestmod=hashlib.sha256
        ).digest()
    ).decode('utf-8')
    
    return hmac.compare_digest(signature, expected)

# In your webhook handler:
@app.route('/webhook', methods=['POST'])
def handle_event():
    signature = request.headers.get('x-twitter-webhooks-signature', '')
    
    if not verify_signature(request.data, signature, CONSUMER_SECRET):
        return 'Invalid signature', 401
    
    # Process event...
```

```typescript TypeScript
import crypto from 'crypto';

function verifySignature(
  payload: Buffer,
  signature: string,
  consumerSecret: string
): boolean {
  const expected = 'sha256=' + crypto
    .createHmac('sha256', consumerSecret)
    .update(payload)
    .digest('base64');
  
  return crypto.timingSafeEqual(
    Buffer.from(signature),
    Buffer.from(expected)
  );
}

// In your webhook handler:
app.post('/webhook', express.raw({ type: '*/*' }), (req, res) => {
  const signature = req.headers['x-twitter-webhooks-signature'] as string;
  
  if (!verifySignature(req.body, signature, CONSUMER_SECRET)) {
    return res.status(401).send('Invalid signature');
  }
  
  // Process event...
});
```
</CodeGroup>

## Best Practices

<CardGroup cols={2}>
  <Card title="Respond quickly" icon="bolt">
    Return 200 OK as soon as you receive the event. Process messages asynchronously.
  </Card>
  <Card title="Handle duplicates" icon="copy">
