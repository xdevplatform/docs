## What the sample demonstrates

Directly relevant to this API:

- **OAuth 1.0a request signing** (required for the Livestream endpoints) — includes a complete, self-contained HMAC-SHA1 implementation for calling `/2/users/:user_id/sources*` and `/2/users/:user_id/broadcasts*`.
- **Complete stream source CRUD** (list/create/rename/delete) against the real API.
- **Full broadcast lifecycle**: create (`NOT_STARTED`), publish with title/`chat_option`, end, list, get, delete.
- **Live chat read access**: the exact 2-step unauthenticated handoff, WebSocket connection, framing, and message parsing.
- A minimal web UI and backend server (Bun + vanilla HTML/JS).

<Note>
**Authentication in the sample:** The sample uses OAuth 1.0a credentials (from environment variables) to call the actual Livestream API endpoints. It also includes an example OAuth 2.0 login flow for its own UI demo, but that is not required to use the Livestream API.
</Note>

## Run it

1. `bun install`
2. Copy `.env.example` → `.env` and fill in the four OAuth 1.0a values (`X_CONSUMER_KEY`, `X_CONSUMER_SECRET`, `X_ACCESS_TOKEN`, `X_ACCESS_TOKEN_SECRET`). Your app must be allow-listed for the Livestream API.
3. `bun run dev`
4. Open `http://localhost:3000`

Use the sample as a working reference for the required OAuth 1.0a signing, request/response shapes, and chat integration. The sample's own `README.md` contains more details on running it and the exact flows it implements.
