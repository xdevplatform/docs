Posts a plain-text chat message to a live broadcast. The message is attributed to the authenticated user. `:id` is the alphanumeric `broadcast_id`.

**Endpoint:** `POST /2/broadcasts/:id/chat`

**OAuth 2.0 scopes:** `broadcast.read`, `broadcast.write`

The broadcast must be currently live. Its chat settings and the authenticated user's eligibility determine whether the user may send a message. For example, chat may be disabled or limited to verified users, subscribers, or users followed by the host.

## Request body

**Required:**

| Field | Type | Description |
|:------|:-----|:------------|
| `text` | string | Message text from 1 through 140 characters. |

## Example

```bash
curl --request POST \
  --url 'https://api.x.com/2/broadcasts/1DxLddgepQbxm/chat' \
  --header 'Authorization: Bearer YOUR_ACCESS_TOKEN' \
  --header 'Content-Type: application/json' \
  --data '{
    "text": "Hello from the X API!"
  }'
```

## Response

```json
{
  "data": {
    "success": true,
    "timestamp": "1772032100000000000"
  }
}
```

`timestamp` is the server timestamp of the message in nanoseconds, returned as a decimal string.

## Errors

- A request returns `400 Bad Request` if the broadcast cannot be found, is not currently live, or the message is invalid.
- It returns `403 Forbidden` if the authenticated user is not permitted to chat in the broadcast.

<Note>
This endpoint sends messages only. To receive chat messages, use the read-access WebSocket flow described in [Accessing Live Chat](/livestream-api/chat/accessing-live-chat).
</Note>
