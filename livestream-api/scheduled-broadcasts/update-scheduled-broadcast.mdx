<Note>
The Livestream Scheduling API is in **closed beta** — breaking changes should be expected. See the [overview](/livestream-api/scheduled-broadcasts/overview).
</Note>

Updates a scheduled broadcast. `:id` is the alphanumeric `broadcast_id`.

**Endpoint:** `PUT /2/broadcasts/scheduled/:id`

**OAuth 2.0 scopes:** `broadcast.read`, `broadcast.write`

<Warning>
**Full replacement, not a partial update.** Before updating, retrieve the current broadcast, change the desired fields, and resend every field that should remain set. Omitting a previously set optional field can clear it. Recurrence cannot be added or changed through this endpoint.
</Warning>

## Request fields

**Required:**

| Field | Description |
|:------|:------------|
| `scheduled_broadcast_id` | The numeric scheduler ID (from create, list, or get). |
| `scheduled_start_ms` | Scheduled start as a Unix timestamp in milliseconds. |
| `scheduled_end_ms` | Scheduled end as a Unix timestamp in milliseconds. |

**Optional:** `title`, `description`, `source_id`, `thumbnail_media_id`, `chat_option`, `locale`, `available_for_replay`, `is_locked`, `manual_publish`, `roll_forward`

Set `roll_forward` to `true` to shift the head of an existing recurring series.

## Example

```bash
curl --request PUT \
  --url 'https://api.x.com/2/broadcasts/scheduled/1DxLddgepQbxm' \
  --header 'Authorization: Bearer YOUR_ACCESS_TOKEN' \
  --header 'Content-Type: application/json' \
  --data '{
    "scheduled_broadcast_id": "2075599796786561024",
    "scheduled_start_ms": "1784007200000",
    "scheduled_end_ms": "1784010800000",
    "title": "Updated product launch",
    "description": "Live from our launch event",
    "source_id": "c47khpz1zuq9",
    "available_for_replay": true,
    "manual_publish": true
  }'
```

## Response

```json
{
  "data": {
    "scheduled_broadcast_id": "2075599796786561024",
    "broadcast_id": "1DxLddgepQbxm",
    "state": "Created",
    "title": "Updated product launch",
    "source_id": "c47khpz1zuq9",
    "scheduled_start_ms": "1784007200000",
    "scheduled_end_ms": "1784010800000",
    "available_for_replay": true,
    "manual_publish": true
  }
}
```

<Tip>
If an update reports incomplete broadcast data, retrieve the current broadcast with [Get a scheduled broadcast](/livestream-api/scheduled-broadcasts/get-scheduled-broadcast) and resend all fields that should remain set.
</Tip>
