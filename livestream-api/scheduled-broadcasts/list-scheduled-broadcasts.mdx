<Note>
The Livestream Scheduling API is in **closed beta** — breaking changes should be expected. See the [overview](/livestream-api/scheduled-broadcasts/overview).
</Note>

Returns scheduled broadcasts owned by the authenticated user.

**Endpoint:** `GET /2/broadcasts/scheduled`

**OAuth 2.0 scope:** `broadcast.read`

## Query parameters

| Name | Description |
|:-----|:------------|
| `max_results` | Number of results to return, from 1 through 100. Default: 20. |
| `oldest_start_time` | Return broadcasts scheduled at or after this Unix timestamp in milliseconds. |
| `newest_start_time` | Return broadcasts scheduled at or before this Unix timestamp in milliseconds. |
| `pagination_token` | Token from the previous response. |

## Example

```bash
curl --request GET \
  --url 'https://api.x.com/2/broadcasts/scheduled?max_results=20' \
  --header 'Authorization: Bearer YOUR_ACCESS_TOKEN'
```

## Response

```json
{
  "data": [
    {
      "scheduled_broadcast_id": "2075599796786561024",
      "broadcast_id": "1DxLddgepQbxm",
      "state": "Created",
      "title": "Product launch",
      "source_id": "c47khpz1zuq9",
      "scheduled_start_ms": "1784000000000",
      "scheduled_end_ms": "1784003600000",
      "manual_publish": true
    }
  ],
  "meta": {
    "result_count": 1,
    "next_token": "NEXT_PAGE_TOKEN"
  }
}
```

If there are no matching broadcasts, the response contains `meta.result_count: 0` and omits `data`.

<Note>
Continue requesting pages until a response contains no data. A `next_token` can be returned for any non-empty page and does not guarantee that another non-empty page exists.
</Note>
