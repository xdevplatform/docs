<Note>
The Livestream Scheduling API is in **closed beta** — breaking changes should be expected. See the [overview](/livestream-api/scheduled-broadcasts/overview).
</Note>

Publishes a scheduled broadcast that was created or updated with `manual_publish: true`. `:id` is the alphanumeric `broadcast_id`.

**Endpoint:** `POST /2/broadcasts/scheduled/:id/live`

**OAuth 2.0 scopes:** `broadcast.read`, `broadcast.write`

## Before calling this endpoint

<Steps>
  <Step title="Enable manual publishing">
    Create or update the broadcast with `manual_publish: true`.
  </Step>
  <Step title="Start sending RTMP video">
    Push video to the broadcast's source.
  </Step>
  <Step title="Confirm the source is active">
    The source must be actively receiving video for the broadcast to publish.
  </Step>
</Steps>

This endpoint has no request body.

## Example

```bash
curl --request POST \
  --url 'https://api.x.com/2/broadcasts/scheduled/1DxLddgepQbxm/live' \
  --header 'Authorization: Bearer YOUR_ACCESS_TOKEN'
```

## Response

```json
{
  "data": {
    "scheduled_broadcast_id": "2075599796786561024",
    "broadcast_id": "1DxLddgepQbxm",
    "state": "Running",
    "source_id": "c47khpz1zuq9",
    "manual_publish": true
  }
}
```

<Warning>
Calling this endpoint for an automatically published schedule is rejected.
</Warning>

## If a go-live request fails

- Confirm `manual_publish` is `true`.
- Confirm the source ID is correct.
- Confirm RTMP video is actively reaching the source.
- Confirm the broadcast has not already ended or been deleted.
