<Note>
The Livestream Scheduling API is in **closed beta** — breaking changes should be expected. See the [overview](/livestream-api/scheduled-broadcasts/overview).
</Note>

Returns one scheduled broadcast. `:id` is the alphanumeric `broadcast_id`, **not** the numeric `scheduled_broadcast_id`.

**Endpoint:** `GET /2/broadcasts/scheduled/:id`

**OAuth 2.0 scope:** `broadcast.read`

## Example

```bash
curl --request GET \
  --url 'https://api.x.com/2/broadcasts/scheduled/1DxLddgepQbxm' \
  --header 'Authorization: Bearer YOUR_ACCESS_TOKEN'
```

## Response

```json
{
  "data": {
    "scheduled_broadcast_id": "2075599796786561024",
    "broadcast_id": "1DxLddgepQbxm",
    "state": "Created",
    "title": "Product launch",
    "source_id": "c47khpz1zuq9",
    "scheduled_start_ms": "1784000000000",
    "scheduled_end_ms": "1784003600000",
    "manual_publish": true
  }
}
```

<Tip>
Poll this endpoint to observe state changes — for example, until the state is `Running` after a [go-live request](/livestream-api/scheduled-broadcasts/go-live), or after the scheduler publishes an automatic broadcast.
</Tip>
