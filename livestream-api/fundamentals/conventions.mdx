## Base URLs and paths

- **Base URL**: `https://api.x.com` (most endpoints); the chat token endpoint uses the legacy `api.twitter.com` host — see [Accessing Live Chat](/livestream-api/chat/accessing-live-chat).
- **Paths**: All management endpoints are under `/2/...`
- **Chat access endpoints**: One X API path (`/1.1/live_video_stream/status/...`) plus one dedicated chat access endpoint (documented in the [chat section](/livestream-api/chat/accessing-live-chat)).

## Requests and responses

- **Request bodies**: JSON (`application/json`).
- **Responses**: JSON. Success responses for create/get/list usually wrap the primary resource in `"source"` or `"broadcast"`.
- **Timestamps**: Milliseconds since Unix epoch. Often returned as strings (e.g. `"created_at_ms": "1678901234567"`); some as numbers.

## Identifiers

| Identifier | Format | Example |
|:-----------|:-------|:--------|
| User IDs | Numeric strings | `"172483972"` |
| Source IDs | Alphanumeric; also used as the `rtmp_stream_key` | `"69rwrye7rnx4"` |
| Broadcast IDs | Alphanumeric; also called `broadcast_id` | `"1AxRnanzLOrxl"` |
| Media keys (for chat/playback) | `28_{numeric}` | `"28_2026675106832068613"` |
