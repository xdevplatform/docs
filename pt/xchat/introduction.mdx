A **Chat API** permite que você envie e receba mensagens diretas **criptografadas de ponta a ponta** no X. Os corpos das mensagens são criptografados no cliente; o X encaminha o texto cifrado e não consegue ler o conteúdo em texto simples. As mensagens também são **assinadas**, para que os destinatários possam verificar o remetente.

---

## O que você precisa no seu app

| Parte | Responsabilidade |
|:------|:-----------------|
| **[Chat XDK](/xchat/xchat-xdk)** | Gerar chaves, criptografar/descriptografar, assinar/verificar, backup seguro de chaves opcional protegido por código de acesso (Python, JS, Rust, Go, C#, Java) |
| **Acesso à X API** | Chaves públicas, chaves de conversa, mensagens, eventos, mídia — via **[XDK](/xdks/python/overview)** (Python/TypeScript) ou HTTPS |
| **Entrega** | [Webhooks ou stream de atividades](/xchat/real-time-events) para eventos ao vivo; API de eventos para histórico |

Siga o **[Guia de introdução](/xchat/getting-started)** para uma implementação completa. Para apenas conceitos, veja o **[Primer de criptografia](/xchat/cryptography-primer)**.

## Como a criptografia funciona (visão geral)

1. Crie pares de chaves de **identidade** e de **assinatura**; armazene as chaves privadas com segurança (backup seguro de chaves ou blob protegido).  
2. **Publique as chaves públicas** para que outros possam trocar chaves de conversa com você e verificar assinaturas.  
3. Compartilhe uma **chave de conversa** publicando cópias criptografadas para cada participante.  
4. **Criptografe e assine** as mensagens de saída; envie apenas o texto cifrado ao X.  
5. **Receba** o texto cifrado via webhooks, stream ou histórico de eventos.  
6. **Verifique e descriptografe** com o Chat XDK.

## Endpoints úteis

Agrupados em **Referência da API** na barra lateral, incluindo:

- Chaves públicas — registrar e buscar  
- Conversas e mensagens — listar/obter conversas, adicionar/rotacionar chaves, eventos, enviar, digitação, leitura, participação em grupo  
- Mídia — enviar e baixar anexos criptografados ([guia](/xchat/media))

## Notas sobre autenticação

Use **OAuth 2.0 user context** com os escopos relacionados a DM (`dm.read`, `dm.write`, além de `users.read` / `tweet.read` conforme necessário; `media.write` para uploads). A atividade do X Chat de um usuário requer a autorização desse usuário. A configuração do backup seguro de chaves é retornada no **seu** registro de chave pública (campo `juicebox_config`) — veja o Guia de introdução.
