## Introdução

Publicar com Localização é o recurso de geolocalização da X API. Esse recurso ajuda a proporcionar uma experiência mais significativa aos usuários, tornando as publicações mais contextuais. Por exemplo, "Fogos de artifício!" é muito mais relevante quando associado a uma localização: "Fogos de artifício!" do bairro SoMa em São Francisco, Califórnia. Estamos ansiosos para trabalhar com o ecossistema de desenvolvedores na criação de ótimas experiências em torno dessa funcionalidade!

## Privacidade do usuário e controle do usuário sobre suas informações de localização

Para garantir uma experiência boa e segura, os desenvolvedores devem ser sensíveis à privacidade do usuário ao lidar com sua localização, além de proporcionar uma consistência básica na experiência "Publicar com Localização". Aqui estão alguns pontos importantes a serem observados para ajudar a proteger a privacidade do usuário e garantir que os usuários compreendam o impacto de suas ações:

## Diretrizes gerais

- Os usuários precisam optar por usar (opt-in) o recurso Publicar com Localização (ativar a localização).
- Os usuários devem dar permissão explícita para que sua localização exata seja exibida com suas publicações. Se o recurso de localização no seu cliente for "persistente" — ou seja, uma vez ativada a localização, ela sempre acompanha uma publicação — talvez seja recomendável exibir apenas os nomes dos lugares (place_ids) e permitir que os usuários optem, adicionalmente, por incluir latitude e longitude exatas por publicação ou como uma configuração alternativa específica.
- Quando você acessa os dados de localização do usuário para criar uma publicação, essas informações não devem ser armazenadas separadamente da publicação sem a permissão explícita do usuário.
- Deve ficar claro para os usuários qual nível de informação de localização, se houver, será exibido em associação com sua publicação. Uma maneira de fazer isso é mostrar um mapa de como essas informações serão exibidas antes que a publicação seja publicada. Se a latitude e a longitude exatas fizerem parte das informações (ou forem a única informação) publicadas pelo usuário no cliente, deve ficar claro para o usuário que suas coordenadas exatas serão publicadas.
- Os usuários devem poder ativar e desativar sua localização a cada vez que compõem uma publicação.
- Você deve seguir os termos da [Developer Policy](/developer-terms/policy.html) do X – incluindo o requisito de que você não pode agregar, armazenar em cache ou armazenar dados de localização e outras informações geográficas da X API, exceto como parte de uma publicação, e não pode usar esses dados de localização ou informações geográficas de forma isolada.

## Exemplos específicos

Exemplo 1: *Sem adição de latitude e longitude explícitas (esta é a UI atual do X Web Client)*

1. Mostre um link para "Adicionar sua localização" quando um usuário compuser uma publicação;
2. Em segundo plano, use a chamada da API do X [reverse_geocode](https://developer.x.com/en/docs/geo/places-near-location/api-reference/get-geo-reverse_geocode.html) e envie a latitude, longitude, precisão do seu posicionamento e a granularidade preferida do usuário (que, por padrão, é "neighborhood");
3. Exiba o nome do lugar padrão (o X tentará ordená-los de uma forma que faça sentido — a ordenação é tal que um cliente do X pode simplesmente exibir a lista sem necessidade de modificação);
4. Permita que o usuário selecione uma localização diferente da lista (por exemplo, mudar de "SoMa, São Francisco" para "South Beach, São Francisco").
5. Importante: Seja transparente com o usuário se você for exibir informações de localização exata. Garanta que o usuário saiba se informações de localização exata serão exibidas.
6. Permita que o usuário escolha ativar ou desativar a localização em cada composição de publicação.

Exemplo 2: *Implementações existentes de geolocalização*

Vários aplicativos cliente do X têm implementado o recurso original de geolocalização (que apenas transmitia lat/lons sem fornecer place_IDs). O comportamento padrão da API imita esse comportamento — ou seja, como desenvolvedor, você não precisa fazer nenhuma alteração no seu aplicativo existente para manter o comportamento atual. De fato, se você simplesmente enviar uma coordenada lat/lon ao X, o X automaticamente fará o geocodificação reversa dessa latitude e longitude e exibirá o place_ID além da lat/lon para áreas nas quais temos dados disponíveis.

Garanta que seja transparente para seus usuários que suas coordenadas exatas estão sendo usadas e exibidas no X. Uma prática recomendada para aplicativos existentes é permitir que os usuários ativem ou desativem sua localização em cada publicação. Você também pode habilitar seus usuários a compartilhar apenas place_IDs em vez de coordenadas exatas como uma configuração padrão "persistente".

## Armazenamento pelo X dos dados de localização do usuário

O X armazenará os dados enviados pelo cliente da seguinte forma:

- O X salvará todos os dados que um usuário opta por exibir publicamente para seus seguidores. Isso é semelhante à forma como o X armazena o carimbo de data/hora que informa quando a publicação foi feita, ou seja, se um usuário publica com suas coordenadas exatas, o X armazenará essa localização exata juntamente com a publicação enquanto ela existir. Os usuários podem limpar os dados de localização das suas publicações clicando no botão "clear my location history" (limpar meu histórico de localização) na página Configurações, conforme descrito [aqui](https://support.twitter.com/articles/122236-how-to-tweet-with-your-location).

## Resumo

A adição de localização a uma publicação agrega uma enorme quantidade de contexto e valor da perspectiva do usuário. No entanto, as implementações de localização no seu cliente devem ter o cuidado de considerar a privacidade e a transparência para o usuário como um aspecto fundamental no design de um produto focado em localização.
