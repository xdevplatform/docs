O arquivo [`skill.md`](https://docs.x.com/skill.md) segue a [especificação agentskills.io](https://agentskills.io/specification) e descreve o que agentes de IA podem *fazer* com a X API. Enquanto o [`llms.txt`](/tools/llms-txt) é um diretório de páginas, o `skill.md` é um resumo de capacidades — ele lista ações específicas, entradas obrigatórias e restrições para que os agentes usem a API de forma mais confiável.

---

## O que está incluído

- **Capacidades** — o que os agentes podem realizar (pesquisar posts, criar posts, gerenciar usuários etc.)
- **Habilidades** — ações específicas organizadas por categoria com os parâmetros obrigatórios
- **Fluxos de trabalho** — procedimentos passo a passo para tarefas comuns
- **Contexto** — informações de referência sobre autenticação, rate limits e arquitetura

```bash
# Fetch the skill file
curl https://docs.x.com/skill.md
```

---

## Endpoints de descoberta

Agentes podem descobrir arquivos de habilidade programaticamente por meio dos endpoints well-known:

```bash
# Discovery endpoint (agent-skills 0.2.0 spec)
curl https://docs.x.com/.well-known/agent-skills/index.json

# Original discovery format
curl https://docs.x.com/.well-known/skills/index.json
```

Você pode adicionar capacidades da X API a qualquer agente que suporte a [CLI de skills](https://www.npmjs.com/package/skills):

```bash
npx skills add https://docs.x.com
```
